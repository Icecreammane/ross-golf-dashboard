## üß† LEARNING STRATEGIES

### The Feynman Technique
1. Pick a concept (e.g., "What is state in React?")
2. Explain it to yourself as if teaching a 5-year-old
3. Identify gaps in your explanation
4. Review material to fill gaps
5. Simplify and use analogies

**With AI:** Ask Claude/Cursor to explain like you're 5, then try to explain it back.

---

### Build-Along Method
1. Watch tutorial and code along
2. Rebuild from scratch without tutorial
3. Add 1-2 unique features
4. Deploy it
5. Share it (Twitter, Reddit, Discord)

**With AI:** Use AI to help add those unique features, not to build the whole thing.

---

### Debugging-First Learning
1. Find a broken CodePen or project
2. Try to fix it yourself
3. Use AI to explain the bug
4. Fix it with AI's guidance
5. Understand WHY it was broken

**With AI:** Ask "Why did this break?" not "Fix this for me."

---

### Daily Consistency > Intensity
- **Better:** 1 hour/day for 30 days
- **Worse:** 10 hours on Saturday, nothing all week

Set up a system:
- Morning: 30 min tutorial
- Evening: 30 min building

## üîß DEBUGGING RESOURCES

### When You're Stuck
1. **Read the error message** (seriously, read it fully)
2. **Google the exact error** ("Uncaught TypeError: Cannot read property")
3. **Check Stack Overflow** (likely someone had same issue)
4. **Ask AI** (Claude, Cursor, ChatGPT)
5. **Rubber duck debug** (explain problem out loud)
6. **Ask community** (Discord, Reddit - show your code)

### Debugging Tools
- **Console.log** - Print values (classic)
- **Chrome DevTools** - Inspect, debug, network tab
- **React DevTools** - See component state/props
- **VS Code Debugger** - Set breakpoints
